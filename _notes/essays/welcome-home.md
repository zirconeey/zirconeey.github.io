上周有读者在公众号后台问我：“那篇讲'热锅冷油'的，你之前是不是发过？我现在炒菜想翻一下，怎么也找不到了……” 我打开后台，往下翻了五分钟。从最近几篇生活之问翻起，越翻心越凉——有些月份更新得密集，有些月份只发了一篇，公众号的时间线像一条混着各种漂浮物的河，越往上游走越费劲。我发现连我自己都得靠 Ctrl+F 关键字才能定位到那篇我亲手写过的文章。

那一刻我才意识到，问题并不是我写得不够多，而是它们没有家。**公众号擅长把一件事推到你面前，但不擅长在你需要它的时候等在原地。** 它适合被读，不适合被找。于是我做了一件酝酿了挺久的事情：把所有公众号文章搬到了一个网站上，并打算从今往后所有内容都同步到这里。这就是你现在收到这篇推文的原因。

先说清楚一件事——**公众号会继续更新，频率不变，写法也不变。** 你订阅着它，照样会在第一时间收到推送，没有任何变化。网站不是来“取代”它的，网站是来“接住”它的。

打个比方，**公众号像河流，网站像图书馆。** 河流送来流动的当下，图书馆收下了所有的过去。河流的好处是它会主动来找你，缺点是水流过去就过去了；图书馆的好处是它什么时候都在，缺点是它不会自己冒出来叫你。两件事原本就是互补的，没必要二选一。

那为什么一定要做一个图书馆？大概有四个让我下决心的理由。**第一是归档。** 前面那个“找一篇旧文要翻五分钟”的尴尬不是个例——公众号的检索能力着实有限，文章一旦沉到下面，就基本和被埋没没什么区别；而在网站上，我可以按分类、按专栏、按关键词把所有文章重新组织一遍，需要哪一篇，三秒就到。**第二是装得下更多。** 公众号对图片张数、视频长度、外部链接都有限制，但有些文章——尤其是科研妙招和学习资料——本来就需要塞 PDF、塞表格、塞跳转，网站没有这些枷锁，写到哪儿就到哪儿。**第三是链接永不过期。** 公众号文章的链接是平台分配的，明天微信改一次规则，今天的链接就不一定还能打开；而网站的链接是我自己的域名，发出去之后基本就是一辈子的事，方便你转发、收藏、嵌进笔记里。**第四是英文版有处可去。** 我同时在维护一个英文学术主页，那里放的是 working paper、CV、学术活动；中文站和英文站各司其职，互不污染。

四条理由听起来都很功能性，但归根到底就是一句话——**公众号让我把文章发出去，网站让我把文章存下来；前者是动作，后者是资产。** 我希望我写过的东西不止服务一次推送，能在更长的时间维度里继续被使用。

中文主站的地址是 [zirconeey.github.io](https://zirconeey.github.io)，建议你点进去看一眼并加进收藏夹；英文学术主页是 [ruizhou03.github.io](https://ruizhou03.github.io)，关心我学术动态的朋友可以直接移步那边。整个网站按“我在干什么”分成四个大分类，每个分类里再设若干个专栏，下面我一个一个介绍。

### 📚 学习资料

我从大学一年级到现在攒下的课程笔记，目前已经积累了一百多份。本科阶段的微观、宏观、计量、博弈论、动态规划、机器学习；研究生阶段的高微、高宏、高计、行为经济、机制设计——能整理出来的我都尽量整理成了 PDF 放上来。每份资料都附一段背景说明，告诉你它对应哪门课、用的什么教材、适合什么样的读者，方便你判断要不要下载。这部分是网站存在感最强的一块，也是我自己回看最多的一块。

### 🔬 科研妙招

写给“想做科研但没人手把手教过自己怎么做”的同行。从 LaTeX 排版的小细节，到论文写作里反复折磨人的小问题，再到学术工具的实用经验——这些零零散散的“妙招”被我集中放在这里。**名字取得不严肃，是因为它们确实就只是妙招——不是教科书，是别人不会专门告诉你但用过的人都知道的小事。**

### ☕ 生活攻略

这个分类里东西最杂，也是最近半年我更新最勤的地方。最大的两个专栏分别是「菜谱专区」和「生活之问」。菜谱专区不只是把菜谱码出来——每道菜都按“备菜 → 主流程 → 并行时间线”三段式写，配齐美国超市买得到的品名、公制单位、关键火候时机；旁边的「厨房小常识」则补充背后的原理：低中高筋面粉到底差在哪、生抽老抽到底分什么场合、“热锅冷油”为什么能防粘。生活之问是另一种风格——挑一个生活里“听过但说不清”的问题（比如视力表上的 5.0 到底什么意思、香蕉变黑了还能不能吃、微波炉到底能不能加热密闭容器），翻文献、查规范，然后用尽量通俗的语言写成一篇五段结构的科普。**这两个专栏是我目前花心思最多、读者反馈也最多的地方。**

### ✍️ 随笔漫谈

不太规律的、写给自己也写给读者的长文。最有连续性的专栏是「生命故事」——每年生日我都会写一篇万字长文，复盘这一年的心境和经历，从十九岁一直写到现在。**这个分类没什么实用性，但我给它留了最大的版面，因为我相信记录本身就是一种意义。** 你要是只看实用文章，可以直接跳过这块；要是好奇我作为一个写公众号的人在想什么，那就推荐从最近一篇生日文开始读。

如果你是头一次访问网站，下面这五件事可以帮你更快上手：

1. **第一次来先收藏首页。** 主站 zirconeey.github.io 的首页有“近期更新”流和四个分类入口，把它存进浏览器收藏栏或者手机主屏，比每次去公众号翻历史快得多。
2. **找老文章用搜索功能。** 每个分类列表页都有一个搜索按钮（点一下展开），关键词不用很精确，“葱花”“老抽”“博弈论”都能搜到对应文章。
3. **重要资料直接下载。** 学习资料里的每份 PDF 都可以一键下载到本地，不用再像在公众号那样靠截图存相册。
4. **想看英文研究跳到英文站。** 中英文是两个独立的站点，关心我学术工作的同行，建议直接去英文站，那边维护得更勤。
5. **新文章会同步推送。** 网站不会主动找你，但公众号会——所以你不需要每天来网站打卡，订阅着公众号就行；网站只是文章的“长期住所”。

如果觉得这些还是有点抽象，下面这张表把公众号和网站最直接的差异列了出来——

| 维度 | 公众号 | 网站 |
|---|---|---|
| 主要角色 | 推送 + 触达新读者 | 归档 + 长期检索 |
| 找老文章 | 历史列表逐篇翻 | 分类 + 搜索 + 永久链接 |
| 内容形态 | 图文为主，受平台限制 | 图文 + 表格 + PDF + 外链 |
| 链接稳定性 | 平台规则可能调整 | 自有域名，长期有效 |
| 互动方式 | 后台留言、点赞、在看 | 文章下方公开评论区 |
| 评论可见性 | 由作者筛选后展示 | 默认全部公开 |
| 更新提醒 | 自动推送 | 主动访问 / 公众号同步 |
| 阅读环境 | 微信内嵌 | 浏览器 / 跨设备 |

一句话总结：**关注公众号，是为了不错过；收藏网站，是为了找得到。** 两者没有谁取代谁，只是站位不同。

写到这里，顺便晒一晒网站现在的家底——一百多份学习资料、四十多篇生活攻略、十几篇科研妙招、几篇随笔漫谈，外加还在持续更新中的菜谱专区。这些数字本身没什么意义，**意义在于它们终于安顿下来，不再每隔一阵就被推送的浪潮冲到后面去。** 我一直相信文字是一种复利——每写一篇，并不只是当天发出来时的那次阅读量；只要它一直在那儿、一直能被搜到，它就会被新读者继续读到、被老读者翻出来重读，一年、两年、十年，都是可能的。河流流过就过了，**而图书馆，是会一直长高的。**

最后是几条邀请，看你愿意挑哪一条：第一是**拍砖**——网站现在的样子是我一个人摸索出来的，肯定有不顺手的地方，发现什么 bug、觉得哪里设计得反人类，欢迎在文章下方评论区直接说出来；第二是**提问题**——「生活之问」专栏的题材八成来自读者，下次你在生活里碰到“这到底是怎么回事”的疑惑，发后台或者评论留言给我，我会挑感兴趣的写成文章；第三是**催更**——主页上有一个公开的“催更”按钮，被催的次数会被记下来，催着催着我就坐不住了；第四是**收藏**——把主页加进书签，下次想找哪篇文章的时候，先来这里翻。

**网站不取代公众号；它把公众号沉淀下来。** 公众号继续是那条流着的河，但所有流过的水，从今天起，都会有一个图书馆把它们一架一架地收好。如果你愿意，**那我们就在这里继续见面。**

——锆铌，2026 年春
