公众号写到现在四年多，回头看才发现一件挺尴尬的事——很多文章里附的下载链接，已经默默死掉了。

最早一批文章里挂的下载链接，全部都用的是北大网盘——在校的时候它是最方便的存储方式。等我去年从北大毕业，账号自动到期，我连自己网盘的账号都登不进去了。这意味着两件事：第一，那些已经失效的链接，我没办法续命；第二，从今往后写新文章再想往里塞 PDF，“阅读原文”那一栏的链接也无处可挂了。

更微妙的是另一种“链接错位”。我有些资料发布之后并不就此定型，过段时间想到改进、看到读者指出错处，我会回去更新本地文件。但公众号文章里挂的链接是当时那一刻的版本，发出去就锁死了，**读者点进去下到的，永远是过时的旧版**。写作不是一锤子买卖，但公众号的链接就是。

其实这个问题早就该解决。**2024 年初我在密歇根交换的那个春假，曾在宿舍里捣鼓了两天，做出了一个网站的雏形**——但后来发现，要靠一个不懂 CS、不会前端的人长期维护实在太难，自己技术的天花板就在那里，每多写一行代码都心虚。雏形也就一直停在了雏形阶段。直到这一年 AI 工具突飞猛进，很多以前要请教半个朋友圈才能解决的事情，现在和 AI 来回聊上几个回合就搞定了。**于是这次的搬家，其实是我酝酿了将近两年才真正动手的事**——你现在收到这篇推文的原因，就是它终于做完了。

先说清楚一件事——**公众号会继续更新，频率不变，写法也不变。** 你订阅着它，照样会在第一时间收到推送，没有任何变化。网站不是来“取代”它的，网站是来“接住”它的。打个比方，**公众号像河流，网站像图书馆**——河流送来流动的当下，图书馆收下了所有的过去。两件事原本就是互补的，没必要二选一。

那为什么一定要再做一个图书馆？大概有三个理由。**第一是归档。** 公众号本身的检索能力着实有限，文章一旦沉到下面，就基本和被埋没没什么区别；网站可以按分类、按专栏、按关键词把所有文章重新组织一遍，需要哪一篇，三秒就到。**第二是装得下更多。** 公众号对图片张数、视频长度、外部链接都有限制；网站没有这些枷锁，写到哪儿就到哪儿——尤其是科研妙招和学习资料里那些塞 PDF、塞表格、塞跳转的内容，终于能完整地躺在一个页面里。**第三是链接永不过期，并且可以更新。** 网站的链接是我自己的域名，不会因为账号过期而失效，今天发出去的链接可以在三年后继续转发；更重要的是——**资料的内容也不再被锁死**：我可以随时在原地更新文件，新读者下载到的永远是最新版本，老读者再回来还是能找到熟悉的入口。这才是我最想要的“长期主义”。

中文主站的地址是 [zirconeey.github.io](https://zirconeey.github.io)，建议你点进去看一眼并加进收藏夹；英文学术主页是 [ruizhou03.github.io](https://ruizhou03.github.io)，关心我学术动态的朋友可以直接移步那边。整个网站按“我在干什么”分成四个大分类，每个分类里再设若干个专栏，下面我一个一个介绍。

### 📚 学习资料

我从大学一年级到现在攒下的课程笔记，目前已经积累了一百多份。本科阶段的微观、宏观、计量、博弈论；研究生阶段的高微、高宏、高计——能整理出来的我都尽量整理成了 PDF 放上来。**这些大多数文件我之前在公众号上陆续都发过**，但它们分散在四年的时间线里很难找；现在它们集中地住在网站里，需要哪份就直接下载，不用再翻历史推送。每份资料我之后还会慢慢补上一段背景说明（对应哪门课、用什么教材、适合什么读者），并且后续也会在网站这边继续更新和订正部分过去文档。**这部分是网站存在感最强的一块，也是我自己回看最多的一块。**

### 🔬 科研妙招

写给“想做科研但没人手把手教过自己怎么做”的同行。从 LaTeX 排版的小细节，到论文写作里反复折磨人的小问题，再到学术工具的实用经验——这些零零散散的“妙招”被我集中放在这里。**很多东西其实是每一个科研人都要走的路，甚至是必经的弯路，但鲜有人坐下来把它们总结出来；也有一些单纯是我觉得好玩、可以替你省下不少时间的小方法。自己淋过雨，所以也想给别人撑把伞。**

### ☕ 生活攻略

这个分类里东西最杂，也是最近半年我更新最勤的地方。最大的两个专栏分别是「菜谱专区」和「生活之问」。

菜谱专区这一块特别契合我自己做饭的偏好——**我做饭比较“学院派”，喜欢可量化的步骤，受不了“少许”“适量”“炒至香味溢出”这种模糊到无法执行的写法。** 而且实际下厨的时候，光是买食材这一关就能折磨倒不少厨房新手，尤其是身在国外的朋友；如果菜谱在最开头就把品名、用量、对应的美国超市叫法、关键火候时机讲清楚，下厨的前后过程就能少掉一大堆不必要的麻烦，做的时候也能更专心。**还有一点我特别想强调——一个成功的下厨留学生，做饭不止要“知其然”，更要“知其所以然”。** 所以菜谱旁边的「厨房小常识」专门讲背后的原理：低中高筋面粉到底差在哪、生抽老抽到底分什么场合、“热锅冷油”为什么能防粘——这些原理懂了，遇到没写在菜谱里的情况也能自己判断。

生活之问是另一种风格——挑一个生活里“听过但说不清”的问题（比如视力表上的 5.0 到底什么意思、香蕉变黑了还能不能吃、微波炉到底能不能加热密闭容器），把它写成一篇五段结构的科普。**不得不说，AI 工具这两年的进步真的让人受益**——我从小到大攒下的“十万个为什么”，很多终于都能在和 AI 聊上几句之后得到答案；而我觉得，**把这些问题和答案以文章的形式存档下来，本身就是一件很有意思的事。** 这两个专栏是我目前花心思最多的地方。

### ✍️ 随笔漫谈

不太规律的、写给自己也写给读者的长文。最有连续性的专栏是「生命故事」——每年生日我都会写一篇万字长文，复盘这一年的心境和经历，从十九岁一直写到现在。**这个分类没什么实用性，但我给它留了最大的版面，因为我相信记录本身就是一种意义。** 你要是只看实用文章，可以直接跳过这块；要是好奇我作为一个写公众号的人在想什么，那就推荐从最近一篇生日文开始读。

如果你是头一次访问网站，下面这五件事可以帮你更快上手：

1. **第一次来先收藏首页。** 主站 zirconeey.github.io 的首页有“近期更新”流和四个分类入口，把它存进浏览器收藏栏或者手机主屏，比每次去公众号翻历史快得多。
2. **找老文章用搜索功能。** 每个分类列表页都有一个搜索按钮（点一下展开），关键词不用很精确，“葱花”“老抽”“博弈论”都能搜到对应文章。
3. **重要资料直接下载。** 学习资料里的每份 PDF 都可以一键下载到本地，不用再像在公众号那样靠截图存相册。
4. **想看英文研究跳到英文站。** 中英文是两个独立的站点，关心我学术工作的同行可以直接去英文站。
5. **公众号和网站今后的分工。** 网站不会主动找你，但公众号会——所以你不需要每天来网站打卡，订阅着公众号就行。**简单说一下今后两边的更新计划：学习资料和科研妙招我应该还是会同步往公众号推送，所以你不会错过那些有“信号性”的更新；而生活攻略——尤其是菜谱、厨房小常识、生活之问这些更日常、更随手的内容——我应该就只在网站这边更新了，所以也欢迎你常来网站逛逛。**

如果觉得这些还是有点抽象，下面这张表把公众号和网站最直接的差异列了出来——

| 维度 | 公众号 | 网站 |
|---|---|---|
| 主要角色 | 推送 + 触达新读者 | 归档 + 长期检索 |
| 找老文章 | 历史列表逐篇翻 | 分类 + 搜索 + 永久链接 |
| 内容形态 | 图文为主，受平台限制 | 图文 + 表格 + PDF + 外链 |
| 链接稳定性 | 账号到期就失效 | 自有域名，长期有效 |
| 文件可更新 | 上传后基本锁死 | 随时替换为最新版本 |
| 互动方式 | 后台留言、点赞、在看 | 文章下方公开评论区 |
| 评论可见性 | 由作者筛选后展示 | 默认全部公开 |
| 更新提醒 | 自动推送 | 主动访问 / 公众号同步 |
| 阅读环境 | 微信内嵌 | 浏览器 / 跨设备 |

一句话总结：**关注公众号，是为了不错过；收藏网站，是为了找得到。** 两者没有谁取代谁，只是站位不同。

写到这里，顺便晒一晒网站现在的家底——一百多份学习资料、四十多篇生活攻略、十几篇科研妙招、几篇随笔漫谈，外加还在持续更新中的菜谱专区。这些数字本身没什么意义，**意义在于它们终于安顿下来，不再每隔一阵就被推送的浪潮冲到后面去。**

需要重点提一句的是——**这一次，你和我的距离比以前都更近了。** 每篇文章的底部都接着一个公开评论区，所有评论默认全部可见、互相可回复，不再像公众号那样必须经我后台筛选才能露面；每篇文章还配了几个表情反应按钮（开心、感谢、有用、收藏⋯⋯随你挑），你点了我都看得到。主页上还专门做了一个“催更”按钮——它的次数是公开记录的，被催久了我就装不下去了。如果你想说点更长的、不太适合公开的话，主页右侧还留了一个“给博主留言”的入口，仅我可见。**简单来说，请你尽情拍砖、提问、催更、点赞——这些都是我更新最大的动力。**

最后是几条邀请，看你愿意挑哪一条：第一是**提问题**——「生活之问」专栏的题材八成来自读者的疑问，下次你在生活里碰到“这到底是怎么回事”的困惑，发后台或者评论留言给我，我会挑感兴趣的写成文章；第二是**收藏**——把主页加进书签，下次想找哪篇文章的时候，先来这里翻；第三是**继续等更新**——公众号那条河流一切如旧，所有流过的水从今天起都会有一个图书馆好好收下。

如果你愿意，**那我们就在这里继续见面。**

——锆铌，2026 年春
